# 做市参数调优指南

> 适用于 Decibel DEX 永续合约做市机器人（Go / Rust 版本通用）

---

## 做市原理简述

机器人每个周期：
1. 在中间价两侧各挂一张限价单（bid 低于中间价，ask 高于中间价）
2. 若成交，赚取 bid–ask 之间的价差
3. 下个周期撤掉旧单，按新中间价重新挂单

**收益来源**：每次双边成交赚一个价差。  
**主要风险**：行情单边快速移动时，一侧被大量成交形成持仓（库存风险）。

---

## 参数详解

### 1. `--spread`（价差，默认 `0.001`）

**含义**：bid 和 ask 之间的总距离，以中间价的比例表示。
```
bid = mid × (1 − spread/2)
ask = mid × (1 + spread/2)
```
`0.001` = 0.1%，即 BTC 价格 $66,000 时，bid ≈ $65,967，ask ≈ $66,033，间距 $66。

**如何选择**

| 市场状态 | 建议范围 | 理由 |
|----------|----------|------|
| 低波动，价格平稳 | `0.001`–`0.002` | 价差小，成交频率高，薄利多销 |
| 正常波动 | `0.002`–`0.005` | 兼顾成交率和保护空间 |
| 高波动（大新闻/暴涨暴跌） | `0.005`–`0.01` | 价差大，减少被"打穿"的风险 |
| 极端行情 | 建议暂停或切 `--dry-run` | 任何价差都可能不够用 |

> **实际经验**：价差设得越小，成交越多但每笔利润越薄；价差越大，每笔利润高但挂单可能长时间没有成交。从 `0.002` 起步，观察成交情况后再调整。

---

### 2. `--order-size`（单笔挂单量，默认 `0.001`）

**含义**：每次挂单的基础资产数量（BTC/USD 市场单位为 BTC）。

**如何选择**

首先看你的账户保证金，一般单笔挂单的名义价值不超过账户权益的 **5%–20%**：

```
建议 order_size ≤ 账户权益 × 20% ÷ BTC当前价格

例：账户 $500，BTC $66,000
建议 order_size ≤ 500 × 0.2 ÷ 66,000 ≈ 0.0015 BTC
```

| 账户规模 | BTC 约 $66,000 时的参考值 |
|----------|--------------------------|
| $100–$300 | `0.0005`–`0.001` |
| $300–$1,000 | `0.001`–`0.002` |
| $1,000–$5,000 | `0.002`–`0.01` |

> 下单量越大，成交后产生的库存越大，风险越高。新手建议从最小档开始。

---

### 3. `--max-inventory`（最大库存，默认 `0.005`）

**含义**：当净持仓的绝对值达到此阈值时，机器人**停止挂新单**（防止库存继续堆积）。

**如何选择**

通常设为 `order_size` 的 **3–10 倍**：
```
max_inventory = order_size × 5
例：order_size = 0.001 → max_inventory = 0.005
```

- 设得太小：稍有成交就停单，机器人频繁暂停
- 设得太大：行情单边时可能损失很大才触发保护

**对应名义风险估算**：
```
最大名义敞口 ≈ max_inventory × 价格
例：0.005 BTC × $66,000 = $330 名义敞口
```

---

### 4. `--skew-per-unit`（库存偏移系数，默认 `0.0001`）

**含义**：每持有 1.0 单位库存，在价差基础上额外增加的半价差。  
库存为正（多头）→ 整体报价下移（卖得更便宜，买得更贵），主动减仓。

**如何选择**

```
偏移量 = 当前库存 × skew_per_unit
例：持仓 0.003 BTC，skew_per_unit=0.0001
偏移 = 0.003 × 0.0001 = 0.00003（0.003%）
```

- `0.0001`：偏移温和，适合波动小的市场
- `0.0005`–`0.001`：偏移激进，库存较大时快速向中间价靠拢，适合波动大的市场

> 这个参数越大，机器人平仓库存的意愿越强，但在行情有利时也会"提前卖飞"。

---

### 5. `--max-margin-usage`（最大保证金使用率，默认 `0.5`）

**含义**：当账户的交叉保证金占用率超过此值时，机器人**暂停挂单**。

**如何选择**

| 风险偏好 | 建议值 |
|----------|--------|
| 保守 | `0.3`–`0.4` |
| 中等 | `0.5`（默认） |
| 激进 | `0.6`–`0.7` |

> 不建议超过 `0.7`，否则市场轻微波动就可能触发强平。

---

### 6. `--refresh-interval`（刷新周期，默认 `20` 秒）

**含义**：两次完整撤单+重新挂单之间的等待时间。

**如何选择**

| 市场状态 | 建议值 | 理由 |
|----------|--------|------|
| 价格变化缓慢 | `30`–`60` 秒 | 节省 gas，减少无效操作 |
| 正常行情 | `15`–`20` 秒（默认） | 平衡频率与成本 |
| 快速波动 | `5`–`10` 秒 | 让报价跟上市场，减少被吃单风险 |

> 周期越短，报价越及时，但链上交易越频繁，gas 消耗越大。

---

### 7. `--cooldown-s`（bid/ask 间隔，默认 `1.5` 秒）

**含义**：同一周期内，挂完 bid 后等待多久再挂 ask。

一般不需要修改，保持 `1.5` 即可。如果网络较慢或频繁超时，可适当增大到 `2`–`3`。

---

### 8. `--auto-flatten` + `--flatten-aggression`

**`--auto-flatten`（默认 `false`）**：库存触达上限时，是否自动挂一张 GTC 减仓单（reduce-only）。

**`--flatten-aggression`（默认 `0.001`）**：减仓单的激进程度，即减仓价格距中间价的偏移比例。
- `0.001`（0.1%）：稍优于市价成交，速度较快
- `0.003`–`0.005`：更保守，等待更好的价格

**建议**：新手先不开启 `--auto-flatten`，手动处理库存。熟悉后再开启。

---

### 9. 自动价差调整（`--auto-spread` 系列参数）

这一组参数解决"价差设得太宽、一直没有成交"与"价差太窄、每笔利润太薄"之间的矛盾，让机器人自动在安全范围内动态调整价差。

#### 工作原理

- **没有成交**：连续 `--spread-no-fill-cycles` 个周期都没有检测到库存变化时，将当前价差缩小 `--spread-step`，直到达到 `--spread-min` 下限。
- **检测到成交**：自动将价差向初始值方向回升 `spread-step × 0.5`，最多恢复到初始 `--spread`。
- **建议模式（默认）**：`--auto-spread` 为 `false` 时，只打印建议日志，不实际修改价差，适合观察学习。

#### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--auto-spread` | `false` | 设为 `true` 才会实际修改价差；`false` 仅打印建议 |
| `--spread-min` | `0.0004` | 价差下限（0.04%）。**不建议设低于此值**，否则波动时可能做亏 |
| `--spread-max` | `0.02` | 价差上限（2%），成交后也不会超过初始 `--spread` |
| `--spread-no-fill-cycles` | `3` | 连续几个周期没成交后才开始收窄价差 |
| `--spread-step` | `0.0002` | 每次调整幅度（0.02%）。收窄时减少 1 步，成交后扩大 0.5 步 |

#### 关于最小价差 `--spread-min = 0.0004`

做市商挂单承担的主要成本是**逆向选择风险**：当有人以你的报价成交时，往往意味着市场已经在向那个方向移动。经验上，BTC/USD 的 Maker 费率通常在 0.02%–0.03%，加上价格滑动预留，**0.04%（0.0004）是合理的安全底线**。设得更低可能导致：

- 每笔成交利润接近零甚至为负
- 库存积累速度大于价差收益

**推荐的自动价差配置（从保守入手）：**
```bash
--spread 0.003 \
--auto-spread \
--spread-min 0.0008 \
--spread-max 0.005 \
--spread-no-fill-cycles 5 \
--spread-step 0.0002
```
这样价差最宽 0.5%，最窄 0.08%，连续 5 个周期没成交才开始收窄，每次缩小 0.02%，节奏较慢，相对安全。

## 常见场景推荐配置

### 场景 A：保守入门（账户 ~$500，低风险）
```bash
--spread 0.003 \
--order-size 0.001 \
--max-inventory 0.004 \
--skew-per-unit 0.0002 \
--max-margin-usage 0.4 \
--refresh-interval 30
```

### 场景 B：标准配置（账户 ~$1,000，正常行情）
```bash
--spread 0.002 \
--order-size 0.001 \
--max-inventory 0.005 \
--skew-per-unit 0.0001 \
--max-margin-usage 0.5 \
--refresh-interval 20
```

### 场景 C：激进配置（账户 ~$3,000，行情稳定）
```bash
--spread 0.0015 \
--order-size 0.003 \
--max-inventory 0.015 \
--skew-per-unit 0.0001 \
--max-margin-usage 0.6 \
--refresh-interval 15 \
--auto-flatten
```

## 上线前必做：干跑验证

任何参数修改后，先加 `--dry-run` 跑几个周期，确认日志中的 bid/ask 价格和 size 符合预期，再去掉该参数正式运行：

```bash
# Go
go run . --spread 0.002 --order-size 0.001 --dry-run

# Rust
cargo run --release -- --spread 0.002 --order-size 0.001 --dry-run
```

观察日志中的输出：
```
INFO placing POST_ONLY order side=BID price=66447 size=0.001 ... dry_run=true
INFO placing POST_ONLY order side=ASK price=66580 size=0.001 ... dry_run=true
```

确认价差、方向、size 都正确后再实盘。

## 关键指标监控

运行时观察以下日志字段：

| 字段 | 含义 | 需要关注的情况 |
|------|------|----------------|
| `inventory` | 当前净持仓 | 长时间偏向单一方向，需调大 `skew_per_unit` |
| `margin_usage` | 保证金使用率 | 接近 `max_margin_usage` 时说明风险偏高 |
| `equity` | 账户权益 | 持续下降说明做市在亏损，需重新审视参数 |
| `open_orders` | 当前挂单数 | 应为 0 或 2，异常时检查网络 |
| `mid` | 中间价 | 为 `null` 时机器人会暂停，等待价格恢复 |
