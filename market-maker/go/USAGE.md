## 1. 一句话解释：这个机器人是干嘛的

想象你是一个**摆摊卖菜的小贩**：

- 市场上有人在喊"我出 100 元买"
- 也有人在喊"我 102 元卖"
- 你就在中间插一脚："我 100.5 元卖，我 101.5 元买"
- 如果有人买了你的货，你立刻低价再买回来；如果有人卖给你，你立刻高价再卖出去
- 你赚的就是这 **1~2 块钱的差价**

这个机器人就是**自动帮你做这个"低买高卖"的小贩**，而且 24 小时不休息。

> ⚠️ **重要**：这不是"稳赚不赔"的！市场剧烈波动时，你可能会亏钱。请务必先在小资金上测试！

## 2. 用机器人之前，你需要准备什么

### 2.1 硬件要求

| 项目 | 要求 |
|------|------|
| 电脑 | Windows / Mac / Linux 都可以，甚至一台闲置的旧笔记本就行 |
| 网络 | 稳定的网络连接（因为机器人需要实时接收市场数据） |
| 运行时间 | 建议 24 小时开机，或者至少在你交易的时间段保持开机 |

### 2.2 账户准备（这是最难的一步，仔细跟着做）

#### 第 1 步：注册 Decibel 账户

1. 打开浏览器，访问 [app.decibel.trade](https://app.decibel.trade)
2. 用邮箱或钱包注册一个账户
3. 完成身份验证（KYC，如果需要的话）

#### 第 2 步：创建 API 密钥

1. 登录 Decibel 后，找到页面上的 **"API"** 或 **"API 管理"** 按钮（通常在个人中心/设置里）
2. 点击 **"创建新 API Key"**
3. 系统会给你一串很长的字符，类似这样：
   ```
   eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```
   **这就是你的 `bearer_token`，复制下来，千万别泄露给别人！**

#### 第 3 步：准备 Aptos 钱包

Decibel 运行在 Aptos 区块链上，你需要一个 Aptos 钱包。

**推荐钱包**（选一个就行）：
- [Petra Wallet](https://petra.app/)（浏览器插件，最简单）

**操作步骤**：
1. 安装钱包插件，创建新钱包
2. **把助记词（12个单词）手抄在纸上，放在安全的地方！** 这是找回钱包的唯一方式
3. 点击钱包中的"接收"，复制你的钱包地址（类似 `0xabc123...`）
4. **导出私钥**：在钱包设置里找到"导出私钥"或"显示私钥"，复制下来
   - 私钥是一串很长的十六进制字符，类似 `0x1234abcd...`
   - **私钥 = 你的银行密码，绝对不要发给任何人！**

#### 第 4 步：创建子账户（Subaccount）

1. 在 Decibel 的交易页面，找到 **"子账户"** 或 **"Subaccount"** 选项
2. 点击 **"创建子账户"**
3. 创建成功后，复制子账户地址（类似 `0xdef567...`）

> 为什么用子账户？就像把交易资金放在"另一个口袋"里，主钱包的钱不会因为机器人出错而被影响。

#### 第 5 步：充值保证金

1. 从交易所（如币安、OKX）买一些 **USDC**
2. 通过 Aptos 网络，把 USDC 提现到你的**子账户地址**
3. 在 Decibel 页面上确认资金到账

**资金建议**：
- 第一次测试：**50~100 USDC** 就够了
- 熟练后：根据你愿意承担的风险决定，不要超过总资产的 10%

## 3. 快速上手：5分钟让机器人跑起来

### 3.1 下载机器人程序

如果你不会编译代码，可以找开发者朋友帮你编译好，或者使用 releases 里的现成程序。

你会得到两个文件：
- `decibel-bot.exe`（Windows）或 `decibel-bot`（Mac/Linux）—— 这是主程序
- `configs/config.yaml` —— 这是配置文件

### 3.2 填写配置文件

找到 `configs` 文件夹里的 `config.yaml`，用**记事本**（Windows）或**文本编辑**（Mac）打开它。

把下面的内容复制进去，然后**把你自己的信息填进去**（替换掉引号里的内容）：

```yaml
env: testnet

decibel:
  # 把你的 API Token 填在这里（从 Decibel 网站复制）
  bearer_token: "你的bearer_token填在这里"
  
  # 把你的钱包私钥填在这里（从 Petra 钱包导出）
  api_wallet_private_key: "你的私钥填在这里"
  
  # 钱包地址会自动从私钥推导出来，留空即可
  api_wallet_address: ""
  
  # 把你的子账户地址填在这里
  subaccount_address: "你的子账户地址填在这里"
  
  # 你想交易的市场，默认是 BTC-USD（比特币对美元）
  market_name: "BTC-USD"
  
  # 推荐人代码（如果有的话，没有就留空）
  builder_code: ""

aptos:
  # Aptos 节点地址，留空时自动根据 env 选择官方节点
  # 如果你有自定义 RPC（如 Alchemy、QuickNode），可在此填写
  fullnode_url: ""

strategy:
  # ========== 杠杆设置 ==========
  # 杠杆就像借钱做生意。
  # 10倍杠杆 = 你有 100 块，机器人会帮你做 1000 块的生意
  # 小白建议：先用 1~3 倍，熟练后再加
  leverage: 1

  # ========== 买卖价差设置 ==========
  # 机器人会在"中间价"两边挂单。
  # bid_spread 是买价偏离中间价的百分比。
  # 0.01 = 1%，如果比特币 100,000 美元，买价就是 99,000 美元
  bid_spread: "0.0001"
  
  # ask_spread 是卖价偏离中间价的百分比。
  # 0.005 = 0.5%，如果比特币 100,000 美元，卖价就是 100,500 美元
  ask_spread: "0.0001"

  # ========== 订单大小 ==========
  # 每笔订单买卖多少比特币。
  # 0.01 = 每次交易 0.01 个 BTC
  # 资金少的话设小一点，比如 0.001
  order_amount: "0.001"

  # ========== 订单档数 ==========
  # 机器人会挂几层单子。
  # 1 = 只在最优价格挂 1 单买 + 1 单卖
  # 3 = 挂 3 单买 + 3 单卖，价格逐层拉开
  order_levels: 1

  # ========== 订单刷新时间 ==========
  # 多久检查一次并更新订单。
  # 30s = 每 30 秒刷新一次
  order_refresh_time: "10s"

  # ========== 成交后冷却时间 ==========
  # 成交后暂停多久再下单，防止过度交易。
  # 60s = 成交后等 60 秒
  filled_order_delay: "60s"

  # ========== 止损设置 ==========
  # 仓位亏到多少百分比时，自动市价平仓止损。
  # 0.03 = 亏 3% 就止损
  stop_loss: "0.03"

  # ========== 止盈设置 ==========
  # 仓位赚到多少百分比时，自动市价平仓止盈。
  # 0.02 = 赚 2% 就止盈
  take_profit: "0.02"

  # ========== 持仓时限 ==========
  # 仓位持有超过这个时间，自动平仓。
  # 30m = 30 分钟
  time_limit: "30m"

  # ========== 批量下单 ==========
  # true = 使用批量下单（省手续费，推荐）
  use_bulk_orders: true
  
  # PostOnly 模式：只挂单，不吃单（省手续费）
  post_only: true

  # ========== 交易相关 ==========
  # 最大未确认交易数
  max_pending_tx: 10
  # 交易确认轮询间隔
  tx_poll_interval: "2s"
  # WebSocket 断线重连等待
  ws_reconnect_delay: "5s"
```

### 3.3 保存配置文件

- Windows：按 `Ctrl + S` 保存
- Mac：按 `Command + S` 保存

### 3.4 运行机器人

#### Windows 用户

1. 按键盘上的 `Win + R`，输入 `cmd`，按回车
2. 输入下面的命令（把路径改成你实际放机器人的地方）：
   ```
   cd C:\Users\你的用户名\Desktop\decibel-mm-go
   ```
3. 运行机器人：
   ```
   .\decibel-bot.exe
   ```

#### Mac 用户

1. 打开"终端"（Terminal）应用
2. 输入：
   ```bash
   cd ~/Desktop/decibel-mm-go
   ```
3. 运行：
   ```bash
   ./decibel-bot
   ```

## 4. 所有参数的大白话解释

### 4.1 网络环境（env）

| 参数 | 可选项 | 小白建议 |
|------|--------|---------|
| `env` | `testnet` / `mainnet` | **先用 `testnet`**（测试网，钱是假的，随便练手），熟练后再换 `mainnet`（主网，真钱） |

### 4.2 账户信息（decibel）

| 参数 | 是什么 | 在哪里找 |
|------|--------|---------|
| `bearer_token` | Decibel 网站的"通行证" | Decibel 网站的 API 管理页面 |
| `api_wallet_private_key` | 你钱包的"密码" | Petra 钱包 → 设置 → 导出私钥 |
| `api_wallet_address` | 你钱包的"账号" | **已从私钥自动推导，无需填写** |
| `subaccount_address` | 专门给机器人用的"子账号" | Decibel 网站 → 子账户页面 |
| `market_name` | 交易什么币 | `BTC-USD`（比特币）、`ETH-USD`（以太坊）等 |

### 4.3 杠杆（leverage）

**杠杆 = 借钱做生意的倍数。**

- 你有 100 USDC，设 1 倍杠杆 → 机器人做 100 USDC 的生意
- 你有 100 USDC，设 5 倍杠杆 → 机器人做 500 USDC 的生意
- 你有 100 USDC，设 10 倍杠杆 → 机器人做 1000 USDC 的生意

| 杠杆 | 收益 | 风险 | 适合谁 |
|------|------|------|--------|
| 1~2 倍 | 赚/亏都慢 | 低 | **新手首选** |
| 3~5 倍 | 中等 | 中等 | 有点经验的用户 |
| 10 倍以上 | 赚/亏都快 | **很高** | 高手，心理素质好 |

> 💡 **新手建议：先用 1~2 倍，跑一周没问题再加。**

### 4.4 买卖价差（bid_spread / ask_spread）

**这是机器人赚差价的核心参数。**

想象比特币当前价格是 **100,000 美元**：

| 参数 | 值 | 含义 | 你的买价 | 你的卖价 |
|------|-----|------|---------|---------|
| bid_spread | 0.01 (1%) | 买价比中间价低 1% | 99,000 | — |
| ask_spread | 0.01 (1%) | 卖价比中间价高 1% | — | 101,000 |

**价差越大 → 单笔赚得越多，但成交越难（没人愿意高价买/低价卖给你）**
**价差越小 → 成交越容易，但单笔赚得越少**

| 市场情况 | bid_spread | ask_spread | 说明 |
|----------|-----------|-----------|------|
| 波动小、流动性好 | 0.002~0.005 | 0.002~0.005 | 价差小，薄利多销 |
| 波动中等 | 0.005~0.01 | 0.005~0.01 | 平衡型 |
| 波动大、流动性差 | 0.01~0.03 | 0.01~0.03 | 价差大，防被套 |

### 4.5 订单大小（order_amount）

**每笔订单买卖多少币。**

假设你总共有 500 USDC 的保证金：

| order_amount | 每笔用多少钱（BTC @ $100k） | 适合总资金 |
|-------------|---------------------------|-----------|
| 0.001 | 100 USDC | 200~500 USDC |
| 0.005 | 500 USDC | 1000~3000 USDC |
| 0.01 | 1000 USDC | 5000 USDC 以上 |

> ⚠️ **不要把所有钱都一次性挂出去！** 至少留 50% 作为安全垫。

### 4.6 订单档数（order_levels）

**机器人同时挂几层单子。**

`order_levels: 1` 时：
```
卖价: 101,000 (1单)
中间价: 100,000
买价:  99,000 (1单)
```

`order_levels: 3` 时：
```
卖价: 101,000 (第1单)
卖价: 102,000 (第2单，拉开一档)
卖价: 103,000 (第3单)
中间价: 100,000
买价:  99,000 (第1单)
买价:  98,000 (第2单)
买价:  97,000 (第3单)
```

| 档数 | 优点 | 缺点 | 适合 |
|------|------|------|------|
| 1 | 简单、资金集中 | 只能吃最优价附近 | **新手推荐** |
| 2~3 | 覆盖更多价格区间 | 资金分散、管理复杂 | 资金较多、波动大时 |

### 4.7 订单刷新时间（order_refresh_time）

**多久重新算一次价格并更新订单。**

| 时间 | 适合情况 | 说明 |
|------|---------|------|
| 10s | 波动极大 | 很耗 gas 费，不推荐新手 |
| 30s | 波动中等 | 比较平衡 |
| 60s | 波动小 | 省手续费，但可能跟不上价格 |

### 4.8 成交后冷却（filled_order_delay）

**成交后暂停多久再下单。**

为什么要冷却？防止机器人"上头了"连续乱交易。

| 时间 | 适合 |
|------|------|
| 30s | 高频交易，经验丰富 |
| 60s | **推荐默认值** |
| 300s (5分钟) | 保守型，减少交易次数 |

### 4.9 止损（stop_loss）

**最多允许亏多少百分比，然后自动割肉。**

例如 `stop_loss: "0.03"` 表示亏 3% 就自动市价平仓。

> 这个非常重要！是保护你不被爆仓的最后一道防线。

| 止损值 | 风格 |
|--------|------|
| 0.01 (1%) | 极保守，经常止损，但亏得少 |
| 0.03 (3%) | **推荐** |
| 0.05 (5%) | 给波动留空间 |
| 0.10 (10%) | 很宽松，可能扛很久 |

### 4.10 止盈（take_profit）

**赚到多少百分比就自动落袋为安。**

例如 `take_profit: "0.02"` 表示赚 2% 就自动平仓。

> 做市策略一般靠频繁小赚积累，所以止盈设小一点很正常。

### 4.11 持仓时限（time_limit）

**一个仓位最多拿多久。**

做市的本质是"快进快出"，如果一个仓位拿了太久，说明市场方向对你不利，该走了。

| 时间 | 说明 |
|------|------|
| 15m | 超短线 |
| 45m | **推荐** |
| 2h | 给趋势一点空间 |

### 4.12 批量下单（use_bulk_orders）

**true** = 把买卖单打包成一个交易提交（省手续费、更原子化）
**false** = 一笔一笔单独下

> **建议保持 true**，除非你在调试问题。

## 5. 三套推荐配置（直接复制粘贴）

### 5.1 保守型（新手首选，保命第一）

适合：第一次用机器人，资金 100~500 USDC，心态偏谨慎

```yaml
env: testnet

decibel:
  bearer_token: "你的token"
  api_wallet_private_key: "你的私钥"
  api_wallet_address: ""
  subaccount_address: "你的子账户地址"
  market_name: "BTC-USD"

strategy:
  leverage: 1
  bid_spread: "0.008"
  ask_spread: "0.008"
  order_amount: "0.001"
  order_levels: 1
  order_refresh_time: "60s"
  filled_order_delay: "120s"
  minimum_spread: "-100"
  stop_loss: "0.02"
  take_profit: "0.015"
  time_limit: "30m"
  use_bulk_orders: true
  post_only: true
  max_pending_tx: 10
  tx_poll_interval: "2s"
```

**特点**：
- 1 倍杠杆，基本不会爆仓
- 价差较大（0.8%），保证每笔都有利润空间
- 订单小（0.001 BTC），试探性交易
- 刷新慢（60秒），减少手续费消耗
- 止损紧（2%），亏一点就跑

---

- 各项参数都是中等水平

### 5.3 激进型（高手，心理素质强）

适合：对市场有判断，资金 3000 USDC 以上，能承受较大回撤

```yaml
env: testnet

decibel:
  bearer_token: "你的token"
  api_wallet_private_key: "你的私钥"
  api_wallet_address: ""
  subaccount_address: "你的子账户地址"

strategy:
  leverage: 5
  bid_spread: "0.003"
  ask_spread: "0.003"
  order_amount: "0.01"
  order_levels: 3
  order_level_spread: "0.003"
  order_refresh_time: "15s"
  filled_order_delay: "30s"
  minimum_spread: "-100"
  stop_loss: "0.05"
  take_profit: "0.03"
  time_limit: "1h"
  use_bulk_orders: true
  post_only: true

```

**特点**：
- 5 倍杠杆，收益放大明显
- 价差很小（0.3%），追求高频成交
- 3 档订单，大资金覆盖
- 刷新快（15秒），紧跟市场
- 止损放宽到 5%，给波动留空间

## 6. 如何选择适合你的参数

### 6.1 第一步：确定你的风险承受力

问自己三个问题：

1. **如果这单子账户的钱全部亏光，我会睡不着吗？**
   - 会 → 用保守型，资金再减一半
   - 有点心疼但可以接受 → 用平衡型
   - 无所谓 → 可以用激进型

2. **我能接受一天内资金波动多少？**
   - 不能超过 5% → 杠杆 1~2，止损 2%
   - 10% 以内 → 杠杆 3，止损 3%
   - 20% 以内 → 杠杆 5，止损 5%

3. **我每天有多少时间看机器人？**
   - 随时能看 → 刷新时间可以短一点
   - 一天看 1~2 次 → 刷新时间长一点，止损紧一点

### 6.2 第二步：根据市场调整

**市场很平静**（价格在很小范围内波动）：
- 缩小价差（bid_spread/ask_spread 从 0.008 降到 0.005）
- 缩短刷新时间（60s → 30s）
- 这是机器人最赚钱的时候！

**市场剧烈波动**（暴涨暴跌）：
- **立刻停掉机器人**（Ctrl+C），或者：
- 拉大价差（0.005 → 0.02）
- 减小订单量（0.01 → 0.001）
- 收紧止损（0.03 → 0.02）
- 或者干脆观望，等平静了再开

**重大新闻前后**（美联储决议、非农数据等）：
- **提前 1 小时关掉机器人**
- 等新闻出来、市场消化后再开

### 6.3 第三步：小步迭代

不要一上来就追求完美参数！建议的顺序：

```
第 1 周：保守型配置 + testnet（假钱练习）
    ↓ 没问题
第 2 周：保守型配置 + mainnet（真钱，小资金）
    ↓ 没问题
第 3~4 周：逐步调整一个参数，观察效果
    ↓ 稳定盈利
第 2 个月：慢慢过渡到平衡型
```

**每次只改一个参数！** 这样你才能知道是哪个参数在影响结果。

## 7. 常见错误与解决办法

### 7.1 机器人打不开/一闪而过

**现象**：双击 `decibel-bot.exe`，黑色窗口闪一下就不见了

**原因**：配置填错了

**解决**：
1. 打开 `configs/config.yaml`
2. 检查所有 `""` 里面是不是都填了内容
3. 检查私钥是不是完整的，没有多复制空格
4. 保存后重新运行

### 7.2 提示 "target market not found"

**现象**：
```
fatal   target market not found     market=BTC-USD
```

**原因**：市场名称写错了，或者这个市场上没开放

**解决**：
- 检查 `market_name` 的大小写，必须和 Decibel 网站上显示的一模一样
- 常见正确写法：`BTC-USD`、`ETH-USD`、`SOL-USD`

### 7.3 提示 "failed to parse aptos signing account"

**现象**：
```
fatal   failed to parse aptos signing account
```

**原因**：私钥格式不对

**解决**：
- 私钥应该是类似 `0x1234abcd...` 的格式
- 确保复制了完整的私钥，没有漏掉开头或结尾
- 不要包含任何空格或换行

### 7.4 提示 "tx failed" 或交易一直失败

**可能原因**：
1. 子账户里没钱了 → 充值 USDC
2. gas 费不够 → 钱包里需要有一些 APT（Aptos 的原生币）作为手续费，0.1~1 APT 就够了
3. 网络拥堵 → 等几分钟再试

### 7.5 机器人挂着但没有任何成交

**检查清单**：
- [ ] 价差是不是设太大了？（比如 5%，别人不愿意接你的单）
- [ ] 订单量是不是太小了？（比如 0.0001，别人看不见）
- [ ] 市场是不是完全没有波动？（等一等）
- [ ] 看看 Decibel 网页上，你的单子实际挂出去没有

### 7.6 机器人成交太频繁，手续费很高

**解决**：
- 拉大价差（spread 从 0.005 改到 0.01）
- 延长刷新时间（30s 改到 60s）
- 延长冷却时间（60s 改到 120s）

## 8. 安全提醒与风险控制

### 8.1 最重要的三条铁律

1. **永远不要把所有钱放进机器人账户**
   - 只用你愿意亏光的资金
   - 建议不超过你总资产的 5%~10%

2. **私钥就是一切**
   - 不要截图发给别人
   - 不要存在微信/QQ/邮件里
   - 手抄在纸上，锁在抽屉里

3. **机器人不是印钞机**
   - 市场单边上涨/下跌时，机器人会亏钱
   - 没有"设置好就不用管"这回事

### 8.2 什么情况必须立刻停掉机器人

出现以下任何一种情况，按 `Ctrl + C` 停止：

- [ ] 市场突然暴涨或暴跌（超过 2% 的瞬时波动）
- [ ] 你看到连续两笔以上的亏损成交
- [ ] 你要离开电脑超过 4 小时，且没设置止损
- [ ] 有重大新闻/公告即将发布
- [ ] 你的网络断开了（机器人可能在下盲单）
- [ ] 感觉"这次不一样"的时候（相信直觉）

### 8.3 亏损了怎么办

**第一步：停掉机器人**（Ctrl+C）

**第二步：不要立刻重新开**
- 情绪化的"报复性交易"会让亏损更大
- 去喝杯水，冷静 30 分钟

**第三步：复盘**
- 看日志，搞清楚为什么会亏
- 是市场方向变了？还是参数设太宽了？

**第四步：调整后再开**
- 降低杠杆
- 收紧止损
- 减小订单量
